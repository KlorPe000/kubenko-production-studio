# Деплой на Railway

## Подготовка к деплою

Проект готов для деплоя на Railway. Все необходимые изменения внесены.

## Шаги для деплоя:

### 1. Создание проекта на Railway
- Зайдите на [railway.app](https://railway.app)
- Нажмите "New Project"
- Выберите "Deploy from GitHub repo"
- Подключите свой репозиторий

### 2. Добавление базы данных PostgreSQL
- В проекте нажмите "New Service"
- Выберите "Database" → "PostgreSQL"
- Railway автоматически создаст базу и установит переменную `DATABASE_URL`

### 3. Настройка переменных окружения
В разделе Variables добавьте:
```
NODE_ENV=production
SESSION_SECRET=ваш-секретный-ключ-для-сессий
TELEGRAM_BOT_TOKEN=ваш-токен-бота (если используете)
TELEGRAM_CHAT_ID=ваш-chat-id (если используете)
```

### 4. Настройка домена (опционально)
- В разделе Settings → Domains
- Можете добавить кастомный домен или использовать предоставленный Railway

## Что изменено для Railway:

✅ **Порт**: Теперь использует `process.env.PORT` для Railway  
✅ **Сессии**: В продакшене используется PostgreSQL вместо памяти  
✅ **Безопасность**: Secure cookies включены в продакшене  
✅ **Конфигурация**: Добавлен `railway.json` для настройки сборки  
✅ **База данных**: Добавлена таблица сессий для продакшена  

## Миграция базы данных

После деплоя выполните:
```bash
npm run db:push
```

Это создаст все необходимые таблицы в PostgreSQL.

## Возможные проблемы:

1. **Ошибка сессий**: Убедитесь что таблица `session` создана в базе
2. **Порт**: Railway автоматически назначает порт через переменную PORT
3. **HTTPS**: Railway автоматически предоставляет SSL сертификаты

## Загрузка файлов

В текущей версии файлы хранятся в памяти. Для продакшена рекомендуется:
- Cloudinary для изображений
- AWS S3 для видео файлов
- Или другое облачное хранилище

Это можно добавить позже без изменения основной функциональности.
